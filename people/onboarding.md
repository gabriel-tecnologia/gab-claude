# Onboarding

Guia para integração de novos membros na área de tecnologia.

**Última atualização**: _preencher_
**Responsável**: _preencher_

---

## Antes do Primeiro Dia

### Responsabilidades do Gestor

- [ ] Criar conta no Google Workspace
- [ ] Adicionar aos canais do Slack (#tech-geral, #tribe-[nome], #squad-[nome])
- [ ] Criar conta no Linear e adicionar ao time
- [ ] Dar acesso ao GitHub (org e repos relevantes)
- [ ] Dar acesso a softwares relevantes:
  - [ ] Slack
  - [ ] Figma
  - [ ] DataDog
- [ ] Agendar 1:1 de boas-vindas

### Equipamento

- [ ] Laptop configurado
- [ ] Acessos de VPN (se aplicável)
- [ ] Licenças de software necessárias

---

## Primeira Semana

### Dia 1 - Contexto

| Horário | Atividade                            | Com quem |
| ------- | ------------------------------------ | -------- |
| 09:00   | Boas-vindas e setup de equipamento   | Gestor   |
| 10:00   | Overview da empresa e produto        | Gestor   |
| 11:00   | Tour pelos sistemas principais       | Buddy    |
| 14:00   | Setup do ambiente de desenvolvimento | Buddy    |
| 16:00   | Leitura: este repositório            | Solo     |

### Dias 2-3 - Imersão Técnica

- [ ] Entender arquitetura geral (ver `tribes/[sua-tribe]/systems/`)
- [ ] Rodar o projeto localmente
- [ ] Fazer primeiro PR (pode ser typo fix)
- [ ] Conhecer cerimônias do squad (ver `sdlc/ceremonies.md`)

### Dias 4-5 - Primeira Contribuição

- [ ] Pegar primeira task no Linear
- [ ] Parear com alguém do squad
- [ ] Participar de code review

---

## Primeiro Mês (30 dias)

### Objetivos

- [ ] Completou pelo menos 3 tasks solo
- [ ] Participou de todas as cerimônias do sprint
- [ ] Conheceu membros de outras tribes
- [ ] Leu toda documentação da sua tribe

### Checkpoints

| Semana | Checkpoint                               | Com quem |
| ------ | ---------------------------------------- | -------- |
| 1      | Como está o setup? Alguma dúvida?        | Buddy    |
| 2      | Feedback inicial, ajuste de expectativas | Gestor   |
| 3      | Revisão de primeiras entregas            | Gestor   |
| 4      | 1:1 de fechamento do primeiro mês        | Gestor   |

---

## Segundo Mês (60 dias)

### Objetivos

- [ ] Autonomia em tasks de complexidade média
- [ ] Contribuiu para documentação
- [ ] Deu feedback sobre processo de onboarding

---

## Terceiro Mês (90 dias)

### Objetivos

- [ ] Autonomia completa no squad
- [ ] Pode ser buddy de novos membros
- [ ] Entende o roadmap da tribe
- [ ] Feedback formal de performance
